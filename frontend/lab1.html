Sobre HTML5

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

    2. El internet es un servidor enorme, mientras que la world wide web es un conjunto de paginas web "

  2. ¿Cuáles son las partes de un URL?

    3. Tiene 4 partes principales: el Protocolo: se refiere al estándar envió de datos, el dominio que indica la IP de la página, la dirrección que indíca el camino para llegar a la página desde el servidor y los parametros que se refieren a la busqueda o query

  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

    4. GET nos permite recuperar datos de una pagina sin modificarlo. El metodo HEAD permite saber si existe el dato a obtener por GET al regresar el encabezado. El método POST permite mandar datos a un servidor. El método PUT permite editar el dato del servidor. El método PATCH permite modifica parcialmente un recurso. El método DELETE elimina un recurso específico"

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

    5. Se debe utilizar el método POST, pues este manda la información de nuestra computadora a el servidor"

  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

    6. El método GET, pues este nos permite visualizar la pagina al utiliazr la URL y representar los recursos en nuestra computadora

  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    7. No ocurre un error, esto nos indica que el query fue exitoso y se redigira al usuario

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    8. No, esto error indica que el query contiene un erro pues no coincide con ningún dominio

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    9. En este caso sí, este codigo indica que aunque el query fue exitoso el error es interno al servidor y es necesario hacer los ajustes necesarios para que se pueda acceder
  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    10. Se refiere a cualquier atributo que ya no se le da soporte porque ha sido remplazado, un ejemplo es "font" que ha sido remplazado por font family y font size

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    11. HTML5 es una versión superior pues tiene un peso semántico mucho mayor y donde se separa el contenido de la estructura, además que la creación de código es más limpia y simple, para declarar el DOCTYPE en HTML es suficiente con escribir DOCTYPE html además de que hay mucho más soporte a etiquetas
  11. ¿Qué componentes de estructura y estilo tiene una tabla?

    12. Tiene muchos entre ellos están: thead, tfoot, tr, tc, tbody, col, colgroup, td y caption

  12. ¿Cuáles son los principales controles de una forma HTML5?

    13. Los principales son: input de texto, area de texto, selección, radio, checkboxes, subida de URL y archivos.

Sobre el ciclo de vida y desarrollo de sistemas de información

  1. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    2. Se refiere a una metodología para aplicable al desarrollo de software y se divide en 7 partes: Planificación, análisis, diseño, desarrollo, integración y pruebas, implementación y mantenimiento.
  2. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    3. Es parte del ciclo de desarrollo de sistemas, pero no sol termina en mantenimiento, sino también tiene una fase de obsolencia y/o remplazo
